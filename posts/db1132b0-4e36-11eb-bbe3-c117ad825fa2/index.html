RE：从零开始的ArchLinux

开始前的准备

制作镜像

  • 下载archLinux镜像

  • 一般下载最新的。由于无法避免丢包等现象，下载完务必校验下载下来的镜像和官网上哈希值是否一致

  • windows

    1. 下载。打开后选择U盘和镜像，磁盘分区格式/引导分区根据实际情况选择mbr/gpt和bios/uefi.
    2. 点击start，写入镜像.如果有弹窗提示,选择dd模式写入.
    3. 写入完成之后要安装的电脑插上U盘重启按F12/F11选择U盘启动.

  • linux

    • 插入U盘查看U盘卷名
    • 使用dd将镜像写入u盘
    • 等待写入完成,电脑从U盘启动.
    1. 选择第一个,boot archlinux

开始安装

硬盘分区

  1. 查看当前硬盘及挂载情况

  2. 对目标磁盘进行分区格式化.

    ←→键选择new在磁盘sda上创建分区.
    首先创建boot分区.first sector默认就行,size in sectors 输入+128M,意为创建128M的分区.

    ←→键选择new在磁盘sda上创建分区.
    首先创建boot分区.first sector默认就行,size in sectors 输入+128M,意为创建128M的分区.

  3. 接下来选择分区格式,linux efi格式代码为ef00,也可以输入L查看更多.
    输入分区名 boot
    分区名为boot格式为linux efi的分区就创建好了

    接着选择剩余的39G的空余空间创建/分区.
    操作和之前一样,不过firest sector还是默认,size in sector那里根据自己需求来划分,我全部划给了/分区,就直接按了回车. /分区格式ext4,代码为默认的8300.

分区完成之后就是这样

选择write进行写入.

硬盘格式化

使用lsblk查看硬盘挂载情况

可以看到128M的boot分区名对应着/dev/sda1,/分区对应着/dev/sda1

  • 挂载 / 分区到/mnt

安装archlinux

换源

文件 /etc/pacman.d/mirrorlist 定义了软件包会从哪个镜像源下载

使用echo和>修改镜像配置文件.

可以看到镜像文件已经修改成了阿里源.

安装

这条命令将从pacmanlist中的源下载并将archlinux安装到/mnt

由于安装是的archlinx正常运行的基本包,所以并不包含以下软件包

  • 管理所用文件系统的用户工具

  • 访问 RAID 或 LVM 分区的工具

  • 含在 linux-firmware 中的额外固件

  • 联网需要的程序

  • 文本编辑器.

  • 等他联网下载安装

到这一步就说明系统安装完毕了.

我们开始进行下一步

配置系统

等安装完成之后，进行archlinux的配置。

  • 生成fstab文件

    fstab文件可用于定义磁盘分区，各种其他块设备或远程文件系统应如何装入文件系统。

  • 换镜像源

    • 由于之前已经修改过了live盘里的镜像配置，所以使用 > 重定向就可以了

  • 切换到新安装的archlinux系统（我们目前还是在live模式下面操作）

  • aur源

    在行末添加

    [archlinuxcn]

    SigLevel = Optional TrustAll

    Server = http://repo.archlinux.org/$arch

    • 配置完后刷新下数据库

    编辑/etc/locale.gen 然后移除需要的地区的注释。

    创建wind用户，并在/home下创建wind用户的home主目录，将其加入wheel组。关于wheel组：

    wheel 组的概念继承自 UNIX。当服务器需要进行一些日常系统管理员无法执行的高级维护时，往往就要用到 root 权限；而“wheel” 组就是一个包含这些特殊权限的用户池；也就是说，如果你不是“wheel”组的成员，就无法取得 root 权限进行一些特权的操作；

安装引导程序

以grub为例,在boot文件夹里创建efi文件夹，再切换到arch live盘，将之前格式化好的boot分区挂载到 /mnt/boot/efi下

再切换到/mnt下安装好的archlinux,安装grub

  • 安装grub到boot/efi
  • 对于bios启动而非efi，则不用分efi分区

生成grub.cfg之后arch的安装就结束了，拔掉U盘退出/mnt下的系统，reboot之后就能进入新安装的archlinux了。

要在系统范围内（对所有用户有效）安装字体，请将文件夹移动到 /usr/share/fonts/ 目录。这些文件需要对每个用户而言都是可读的，使用 chmod 来设置合理的权限 (比如，文件至少为 0444 ，而目录至少为 0555)。
